  <action>
1. Create `web/app/api/auth/register/route.ts` -- POST handler:
   - Parse JSON body: `{ inviteCode, username, password }`
   - Validate all three fields are present (400 if missing)
   - Validate username: 3-20 chars, alphanumeric + underscores only (400 if invalid)
   - Validate password: minimum 8 chars (400 if too short)
   - Call `validateInviteCode(inviteCode)` -- return 400 with error if invalid/used
   - Call `findUserByUsername(username)` -- return 400 "Username already taken" if exists
   - Hash password: `await hash(password, { memoryCost: 19456, timeCost: 2, parallelism: 1 })` using `@node-rs/argon2`
   - Generate userId with `nanoid()`
   - Call `createUser({ id: userId, username, passwordHash, createdAt: new Date().toISOString() })`
   - Call `claimInviteCode(inviteCode, userId)`
   - Create session: `const session = await getSession(); session.userId = userId; session.username = username; await session.save();`
   - Return `Response.json({ ok: true })`
   - CRITICAL: Always `await session.save()` BEFORE returning the Response.
   - Import `hash` from `@node-rs/argon2` (NOT from `argon2`)

2. Create `web/app/api/auth/login/route.ts` -- POST handler following 02-RESEARCH.md Login Route example:
   - Parse JSON body: `{ username, password }`
   - Validate both fields present (400 if missing)
   - Call `findUserByUsername(username)` -- return 401 "Invalid username or password." if not found
   - Call `verify(user.passwordHash, password)` from `@node-rs/argon2` -- return 401 same error if invalid
   - SECURITY: Use identical error message for "not found" and "wrong password" to prevent username enumeration
   - Create session, save, return `{ ok: true }`

3. Create `web/app/api/auth/logout/route.ts` -- POST handler:
   - Get session via `getSession()`
   - Call `session.destroy()` -- this removes the cookie
   - Return `Response.json({ ok: true })`
  </action>
