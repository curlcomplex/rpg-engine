1. Create root `package.json` with npm workspaces:
   ```json
   {
     "private": true,
     "workspaces": ["packages/*", "mcp-server", "web"]
   }
   ```

2. Scaffold Next.js in `web/` directory. Run from RPG root:
   ```bash
   npx create-next-app@latest web --typescript --tailwind --eslint --app --import-alias "@/*" --turbopack --yes
   ```
   IMPORTANT: Do NOT use `--src-dir` flag. Paths should be `web/app/`, `web/lib/`, not `web/src/app/`. This matches the documented architecture.

3. Install auth dependencies in `web/`:
   ```bash
   cd web && npm install iron-session @node-rs/argon2 nanoid @anthropic-ai/sdk
   ```
   CRITICAL: Use `@node-rs/argon2`, NOT `argon2`. The original `argon2` package has ABI issues with Node.js 23+ and will fail to build on Node v25.6.0. `@node-rs/argon2` is a drop-in replacement with prebuilt binaries via napi-rs.

4. Add `transpilePackages` to `web/next.config.ts`:
   ```typescript
   const nextConfig = {
     transpilePackages: ['@rpg-engine/core'],
   };
   export default nextConfig;
   ```

5. Generate environment secrets and write to `web/.env.local`:
   ```
   SESSION_SECRET=<generate with crypto.randomBytes(32).toString('hex')>
   ENCRYPTION_KEY=<generate with crypto.randomBytes(32).toString('hex')>
   DATA_DIR=./data
   ```
   Generate real random values, not placeholders.

6. Update root `.gitignore` to add:
   ```
   .env*
   .env.local
   data/
   web/.next/
   web/node_modules/
   ```
   Keep all existing entries. Add these at the end.

7. Create the data directory structure:
   ```bash
   mkdir -p data/users
   ```

8. Verify the scaffold works: `cd web && npm run dev` should start without errors. Kill the dev server after confirming startup.

  <action>
1. Create `web/lib/constants.ts`:
   - Export `DATA_DIR` = `process.env.DATA_DIR || path.join(process.cwd(), '..', 'data')` (resolve relative to RPG root, not web/)
   - Export `USERS_DIR` = `path.join(DATA_DIR, 'users')`
   - Export `CODES_PATH` = `path.join(DATA_DIR, 'invite-codes.json')`
   - Use `node:path` for joins

2. Create `web/lib/auth.ts` following the exact pattern from 02-RESEARCH.md Pattern 1:
   - Export `SessionData` interface with `userId?: string` and `username?: string`
   - Export `sessionOptions` (cookieName: `rpg-session`, httpOnly, secure in production, sameSite: lax, maxAge: 14 days, path: /)
   - Export `async function getSession()` that calls `getIronSession<SessionData>(await cookies(), sessionOptions)`
   - CRITICAL: `cookies()` from `next/headers` is async in Next.js 16. MUST be `await`ed.
   - Also export `sessionOptions` for use by middleware (which imports iron-session directly)

3. Create `web/lib/crypto.ts` following 02-RESEARCH.md Pattern 3:
   - `encrypt(plaintext: string): string` -- AES-256-GCM, random 12-byte IV, returns `iv:authTag:ciphertext` (all hex)
   - `decrypt(encryptedData: string): string` -- reverses the above
   - `getEncryptionKey()` reads `process.env.ENCRYPTION_KEY`, validates it is 64-char hex string
   - Use `node:crypto` only (createCipheriv, createDecipheriv, randomBytes)
   - NEVER use a static IV. Always `randomBytes(12)` per encryption.

4. Create `web/lib/users.ts` following 02-RESEARCH.md User Lookup example:
   - Export `UserProfile` interface: `{ id: string, username: string, passwordHash: string, createdAt: string }`
   - Export `async findUserByUsername(username: string): Promise<UserProfile | null>` -- scan USERS_DIR
   - Export `async createUser(profile: UserProfile): Promise<void>` -- create `{USERS_DIR}/{id}/profile.json` and `worlds/` subdirectory
   - Import `USERS_DIR` from constants.ts

5. Create `web/lib/invite-codes.ts` following 02-RESEARCH.md Pattern 6:
   - Export `async validateInviteCode(code: string): Promise<{ valid: boolean; error?: string }>` -- reads JSON, checks existence and used_by
   - Export `async claimInviteCode(code: string, userId: string): Promise<void>` -- sets used_by and claimed_at, writes atomically
   - Import `CODES_PATH` from constants.ts

6. Create `web/lib/anthropic.ts` following 02-RESEARCH.md Pattern 4:
   - Export `async validateApiKey(apiKey: string): Promise<{ valid: boolean; error?: string }>` -- calls `client.models.list({ limit: 1 })`, returns validation result
   - Handle 401 (invalid key), 403 (no access), and network errors with user-friendly messages
   - Uses `@anthropic-ai/sdk` (Anthropic constructor)

7. Create `web/middleware.ts` following 02-RESEARCH.md Pattern 5:
   - Import `getIronSession` from `iron-session` and session config from `@/lib/auth`
   - Define `publicPaths = ['/login', '/register']`
   - Allow public paths and `/api/auth/` routes through
   - Read session: `getIronSession<SessionData>(await cookies(), sessionOptions)`
   - If no `session.userId`, redirect to `/login`
   - Export config matcher: `['/((?!_next/static|_next/image|favicon.ico).*)']`
   - IMPORTANT: Middleware can only READ sessions, not write them.

8. Create `scripts/generate-invite-codes.ts`:
   - Generates 15 invite codes in format `XXXX-XXXX-XXXX` using nanoid
   - Writes to `data/invite-codes.json` as `{ codes: { [code]: { used_by: null, created_at: ISO } } }`
   - Run it: `npx tsx scripts/generate-invite-codes.ts`

9. Seed the invite codes by running the script so `data/invite-codes.json` exists with 15 unused codes.
